1) STL 라이브러리에서 컨테이너 클래스, 이터레이터 클래스, 
  알고리즘 함수의 역할을 각각 설명하시오.
 
  컨테이너 클래스는 데이터를 효율적으로 저장하고 관리하는 클래스로 다양한 
 자료구조를 제공하며, 데이터를 삽입, 삭제, 검색하는 데 필요한 기본적인 
 기능을 제공하고 각 자료구조는 용도에 맞게 최적화된 성능을 보인다.

  이터레이터 클래스는 컨테이너의 요소에 접근하고 순차적으로 탐색할 수 있게 
 해주는 객체로, 포인터처럼 동작하며 컨테이너의 내부 구조와 관계없이 작업할 
 수 있게 한다. 반복문을 사용해 컨테이너의 요소를 효율적으로 순회할 수 있다.

  알고리즘 함수는 컨테이너와 이터레이터를 활용하여 데이터를 처리하는 고수준 
 연산을 제공하며, 함수형 스타일로 설계되어 사용자 정의 함수나 람다 표현식을 
 인자로 받아 재사용 가능한 코드를 작성할 수 있다.

2) 포인터와 iterator의 공통점과 차이점은 각각 무엇인가?

  공통점: 포인터와 이터레이터는 모두 데이터를 순차적으로 접근하는 객체로, 
 요소의 주소를 저장하고 해당 위치에서 데이터를 읽거나 수정할 수 있다.

  차이점: 포인터는 주로 배열이나 동적 메모리 할당에서 사용되며, 메모리 주소를 
 직접 다룬다. 성능 면에서 유리할 수 있지만, 잘못된 주소 접근으로 버그를 
 일으킬 수 있다.
 이터레이터는 컨테이너의 내부 구조와 관계없이 작업할 수 있어 코드의 유연성 
 및 안정성을 높이고 포인터보다 더 높은 수준의 추상화를 제공하여, 컨테이너에 
 의존적인 작업에서 더 안전하고 일관된 동작을 보인다.

  따라서 포인터는 메모리 접근이 빠르고 직접적인 반면, 이터레이터는 더 높은 
 수준의 추상화와 안전성을 제공하며, 다양한 컨테이너와 호환성에서 장점을 가진다.

3) vector 클래스의 erase, insert 멤버함수의 매개변수, 
  리턴값, 기능을 설명하시오.

  vector 클래스의 erase와 insert 멤버 함수는 벡터 내 요소를 삭제하거나 
 삽입하는 기능을 제공한다.

  erase 함수는 벡터에서 특정 위치나 범위의 요소를 제거하고 매개변수는 
 삭제할 요소의 위치를 가리키는 단일 이터레이터나 삭제할 범위의 시작과 끝을 
 나타내는 두 이터레이터를 받아 삭제된 요소의 다음 위치를 가리키는 
 이터레이터를 리턴하고 벡터의 크기는 감소한다

  insert 함수는 지정된 위치에 하나 또는 여러 개의 값을 삽입합니다. 
 매개변수로는 삽입할 위치를 나타내는 이터레이터와 삽입할 값 또는 값의 개수를 
 받아 삽입된 요소의 위치를 가리키는 이터레이터가 리턴되며, 벡터의 크기는 
 증가한다.

  따라서 두 함수 모두 벡터의 내부 구조를 변경하고, 
 요소들의 순서를 재조정하는 기능을 수행한다.
